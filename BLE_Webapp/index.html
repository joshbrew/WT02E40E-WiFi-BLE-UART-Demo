WT02E40E BLE + Wi-Fi Console

Browser talks to the WT02E40E over Web Bluetooth. Node hosts this page and listens for Wi-Fi/UDP packets from the board. Open with http://localhost:8080. The board advertises as WT02E40E-CMD.

Command reference

      

        All supported command shapes
        
    
          
  • status or wt status, prints full mode/BLE/Wi-Fi uptime status. status json returns structured JSON-ish state.
    • 
          
  • id, version, config, config json, print device identity, firmware version, and runtime config.
    • 
          
  • mode idle, disables BLE advertising and Wi-Fi request state. From BLE, this responds first and then disconnects.
    • 
          
  • mode ble, BLE only. This is the default boot mode.
    • 
          
  • mode wifi, Wi-Fi only.
    • 
          
  • mode both, BLE plus Wi-Fi.
    • 
          
  • ble on, ble off, ble status. From BLE, ble off responds first and then disconnects.
    • 
          
  • ble name, ble name set <name>, or top-level name set <name>, changes the advertised BLE name at runtime.
    • 
          
  • wifi on, wifi off, wifi status, wifi reconnect.
    • 
          
  • wifi scan, wifi scan json, wifi scan start, wifi scan last [json], and wifi scan item <n> json, scan nearby APs with small BLE-safe responses. wifi scan full json returns one framed response stream.
    • 
          
  • wifi scan connect <index> <password> [wpa2|auto|wpa3] and wifi scan open <index>, use a scan result to configure credentials.
    • 
          
  • wifi cmd on, wifi cmd off, wifi cmd status, and wifi cmd port [port], controls and live-rebinds the UDP command listener.
    • 
          
  • discovery on, discovery off, discovery status, controls UDP discovery beacons to port 5000.
    • 
          
  • boot status, boot mode <idle|ble|wifi|both>, controls saved boot radio mode. Use config save to persist.
    • 
          
  • bridge status, bridge target <ip> <port>, bridge ble|uart|wifi|all on|off, bridge send <msg>, controls bridge forwarding rules.
    • 
          
  • ping, ping uart, ping ble, ping wifi <ip> <port>, tests latency/control paths.
    • 
          
  • fw status, fw reboot [delay], fw reboot bootloader [delay], firmware/reboot hooks.
    • 
          
  • Optional request IDs: prefix a command with #42, for example #42 status. Responses preserve the ID.
    • 
          
  • Delayed radio cutoff: ble off 5s, wifi off 5s, mode ble 5s.
    • 
          
  • wifi cred list, lists stored runtime credentials.
    • 
          
  • wifi cred set <ssid> <password> [wpa2|auto|wpa3], clears runtime credentials and stores/selects one secured network.
    • 
          
  • wifi cred add <ssid> <password> [wpa2|auto|wpa3], adds another secured network.
    • 
          
  • wifi cred open <ssid>, stores/selects an open network.
    • 
          
  • wifi cred forget <ssid>, removes one stored runtime credential.
    • 
          
  • wifi cred clear, clears runtime credentials and falls back to compiled defaults.
    • 
          
  • tx ble <message>, sends a message over the BLE TX notify characteristic if subscribed.
    • 
          
  • tx uart <message>, writes a message to the firmware UART TX path.
    • 
          
  • tx wifi <ipv4> <port> <message>, sends a UDP packet over Wi-Fi.
    • 
          
  • tx both <ipv4> <port> <message>, sends using the combined TX path implemented in firmware.
    • 
          
  • Arguments can be quoted with "..." or '...'. Use backslash escapes for quotes/backslashes inside an argument.
    •
